WORKFLOW THROUGH FILES:
=======================

1. User Uploads Cloth (Frontend):
   components/ClothUpload.tsx
        ↓ (uses)
   lib/api.ts → POST to backend
        ↓
   main.py receives request
        ↓
   services/cloth_preprocessor.py processes image
        ↓
   Saves to temp/cloth_processed/

2. User Uploads Photo (Frontend):
   components/PersonUpload.tsx
        ↓ (uses)
   lib/api.ts → POST to backend
        ↓
   main.py receives request
        ↓
   services/person_preprocessor.py processes image
        ↓
   Saves to temp/person_processed/

3. Run Virtual Try-On (Frontend):
   components/TryOnResult.tsx
        ↓ (uses)
   lib/api.ts → POST to backend
        ↓
   main.py receives request
        ↓
   services/tryon_service.py runs AI model
        ↓
   Saves result to results/
        ↓
   Returns path to frontend
        ↓
   components/TryOnResult.tsx displays image

STARTUP FILE EXECUTION ORDER:
==============================

Terminal 1 (Backend):
1. cd D:\Look1nce\backend
2. venv\Scripts\activate    (loads venv environment)
3. python main.py           (runs main.py)
   - Imports services/__init__.py
   - Imports services/cloth_preprocessor.py
   - Imports services/person_preprocessor.py
   - Imports services/tryon_service.py
   - Starts Uvicorn server
   - Listens on port 8000

Terminal 2 (Frontend):
1. cd D:\Look1nce\frontend
2. bun dev                  (runs Next.js dev server)
   - Reads next.config.js
   - Reads tsconfig.json
   - Compiles app/layout.tsx
   - Compiles app/page.tsx
   - Compiles all components/*.tsx
   - Processes globals.css with Tailwind
   - Starts dev server
   - Listens on port 3000

HELPFUL TIPS:
=============
1. Always activate venv before running backend
2. Keep both terminals open while using app
3. Backend must start before frontend for full functionality
4. Check terminal logs if something doesn't work
5. Browser console (F12) shows frontend errors
6. Backend terminal shows backend errors
7. Delete temp files periodically to save space
8. Model downloads automatically on first run
9. Results are saved permanently until manually deleted
10. You can run multiple try-ons without restarting
